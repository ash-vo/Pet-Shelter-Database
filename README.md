# Pet Shelter Database

## Overview
This project involves designing and implementing a relational database for a pet shelter to manage animals, adopters, and adoption records. The database supports efficient data storage, retrieval, and analysis of shelter operations.

## Features
- Relational database schema with normalized tables  
- Primary and foreign key constraints to ensure data integrity  
- SQL queries for data retrieval and analysis  
- Insights into adoption trends and shelter activity  

## Database Design
The database includes key entities such as:
- Animals  
- Adopters  
- Adoption Records  

Relationships between tables are structured to maintain consistency and support complex queries.

## SQL Techniques Used
- JOIN operations to combine data across tables  
- Aggregation functions (COUNT, GROUP BY) for analysis  
- Filtering and sorting for targeted queries  
- Normalization to reduce redundancy and improve efficiency  

## Example Queries
- Retrieve available animals  
- Analyze adoption trends over time  
- Identify most adopted animal types  

## Files
- `schema.sql` – database structure and table creation  
- `queries.sql` – SQL queries for analysis  
- `sample-data.sql` – example data for testing
